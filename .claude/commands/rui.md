# Refine UI Block (/rui)

You are helping to refine and update existing shadcn/ui components or blocks.

## Guidelines:

1. **Identify target**: Determine which component needs refinement
2. **Understand changes**: Clarify what needs to be improved or modified
3. **Maintain compatibility**: Ensure changes don't break existing usage
4. **Improve quality**: Enhance code quality, accessibility, or performance
5. **Test considerations**: Note any testing implications

## Common Refinements:

### Styling Updates:
- Adjust colors, spacing, or typography
- Improve responsive behavior
- Add variants or size options

### Functionality Enhancements:
- Add new props or features
- Improve state management
- Enhance user interactions

### Code Quality:
- Refactor for better readability
- Optimize performance
- Add error handling

### Accessibility:
- Improve ARIA labels
- Enhance keyboard navigation
- Increase contrast ratios

## Process:

1. Read the existing component code
2. Identify the specific areas to refine
3. Make targeted improvements
4. Test the changes
5. Update documentation if needed

## Output Format:

- Modified component code
- Explanation of changes made
- Migration notes if API changed
- Updated usage examples

Remember: This is an **Astro + React** project. Consider impacts on both .tsx and .astro file usage.
